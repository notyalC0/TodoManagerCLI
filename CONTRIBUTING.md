By participating in this project, you agree to maintain a respectful and welcoming environment for everyone.

<h2 style="color: #89DDFF;">🛠️ How Can I Contribute?</h2>

### Reporting Bugs
* Check the **Issues tab** to see if the bug has already been reported.
* If not, create a new issue. Include:
    * A clear title and description.
    * Steps to reproduce the bug.
    * Your terminal environment (Windows CMD, PowerShell, Zsh, etc.).

### Suggesting Enhancements
We are always open to new ideas for the terminal UI or new commands! Open an issue with the tag <span style="color: #C792EA;"><b>enhancement</b></span>.

### Pull Requests
1. **Fork** the repository.
2. Create a **Branch**: `git checkout -b feature/AmazingFeature`
3. **Commit** your changes: `git commit -m 'Add some AmazingFeature'`
4. **Push** to the branch: `git push origin feature/AmazingFeature`
5. Open a **Pull Request**.

<h2 style="color: #89DDFF;">💻 Development Setup</h2>

### Prerequisites
* **Dart SDK** (Version 3.0.0 or higher).
* Terminal with **Unicode support** (for icons and progress bars).

### Local Setup

```bash
# Clone the repository
git clone [https://github.com/seu-usuario/TO-DO-LIST-DART-CLI.git](https://github.com/seu-usuario/TO-DO-LIST-DART-CLI.git)

# Enter the directory
cd TO-DO-LIST-DART-CLI

# Get dependencies
dart pub get

# Run in dev mode
dart run bin/main.dart

    Follow the official Dart Style Guide.

    Use dart format . before committing your code.

    Ensure your code doesn't have any linter warnings (run dart analyze).

If you add a new feature, please try to add a corresponding test in the test/ folder.

To run existing tests:
Bash

dart test
